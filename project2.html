CODELENS-X

Project Overview

A systems-level code analysis pipeline that integrates LLVM’s clang-tidy static analyzer with a Python processing layer and LLM-based explanation engine. The system parses low-level C/C++ diagnostics, structures them into semantic issue objects, and generates human-readable debugging insights including severity classification, impact analysis, and fix suggestions. Focused on improving interpretability of compiler-level warnings for safer and more maintainable C/C++ code.

GitHub Repository

Key Features

  • Integrates LLVM clang-tidy to perform static analysis on C/C++ source code
  • Parses raw compiler diagnostics into structured issue objects (file, line, rule, severity, message)
  • Implements a multi-stage pipeline: static analysis → parsing → categorization → AI explanation
  • Uses LLMs to generate clear explanations of issues including cause, impact, and fixes
  • Applies deterministic severity mapping to standardize clang-tidy warnings and errors
  • Generates structured, developer-friendly reports for fast debugging and issue review
  • Provides an interactive Streamlit UI for real-time code analysis and visualization

How It Works

The system takes C/C++ source code as input through a Streamlit interface and writes it to a temporary file for processing.

LLVM clang-tidy is executed on the file to perform static analysis and generate low-level compiler diagnostics including warnings and errors.

A Python parsing layer processes the raw clang-tidy output, extracting structured issue data such as file location, rule ID, severity, and diagnostic message.

Each extracted issue is sent to a Large Language Model (GPT-4.1-mini), which generates structured explanations including cause, impact, and recommended fixes.

The final results are aggregated and displayed in a Streamlit dashboard as a structured debugging report with summaries and detailed issue breakdowns.

Challenges & Solutions

Clang-tidy output contained inconsistent formatting across different warning types
Solution: Developed a robust Python parsing layer using regex and structured extraction logic to normalize diagnostic outputs into consistent issue objects.

PowerShell output redirection caused corrupted or incomplete clang-tidy logs
Solution: Switched to CMD-based execution with proper stderr redirection to ensure clean and complete tool output capture.

Regex patterns initially failed due to variability in file paths and diagnostic formats
Solution: Refined parsing logic to handle Windows paths, multi-line warnings, and clang-tidy’s structured output format.

AI responses were inconsistent and sometimes deviated from expected format
Solution: Enforced strict system prompt constraints to ensure deterministic, structured explanation output.

Multiple issues needed scalable processing instead of single-instance handling
Solution: Implemented iterative issue processing pipeline to handle batch clang-tidy diagnostics efficiently.

Integration complexity between static analysis, parsing, and UI layers
Solution: Refactored system into modular pipeline architecture separating analysis, processing, and presentation layers.
